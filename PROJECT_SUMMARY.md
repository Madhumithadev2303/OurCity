# 🏙️ CrowdComplaint - Smart City Issue Reporting Platform

**CrowdComplaint** is a full-stack platform empowering citizens to report infrastructure issues, verify them through community upvoting, and track resolution by municipal workers.

## 🚀 Key Features

### 1. 👥 Role-Based Access
- **Citizen**: Report issues, track status, and upvote urgent problems.
- **Supervisor**: View analytics, assign tasks to workers, and monitor city health.
- **Worker**: View assigned tasks, update status, and upload "Proof of Work" (After photos).

### 2. 🗺️ Live Incident Map
- Interactive map view showing all issues across the city.
- Color-coded pins for status (Orange = Pending, Green = Resolved).
- **Seeded Data**: Pre-loaded with realistic complaints and locations for immediate testing.

### 3. 👍 Community Upvoting
- Citizens can upvote issues to signal urgency.
- Supervisors can sort complaints by "Most Upvoted" to prioritize resources.

### 4. 📊 Analytics Dashboard
- Visual **Bar Charts** showing complaints by category (Roads, Sanitation, etc.).
- Overview stats for Total, Pending, and Resolved cases.

### 5. 📸 Visual Evidence
- **Before/After** Workflow: Citizens upload the "Before" photo, Workers upload the "After" photo upon resolution.

---

## 🛠️ Technology Stack

- **Frontend**: React.js, Vite, Leaflet (Maps), Recharts (Analytics), CSS Modules (Glassmorphism UI).
- **Backend**: Java Spring Boot, REST API.
- **Database**: H2 In-Memory Database (Data resets on restart).
